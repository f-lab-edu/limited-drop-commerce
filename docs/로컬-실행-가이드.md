# 로컬 실행 가이드

## 사전 요구사항

| 도구 | 버전 | 확인 명령어 |
|------|------|-------------|
| Java | 21 이상 | `java -version` |
| Docker | 최신 권장 | `docker --version` |
| Docker Compose | V2 이상 | `docker compose version` |

---

## 1. 환경 변수 설정 (.env)

프로젝트 루트에 `.env` 파일을 생성한다.  
이 파일은 `.gitignore`에 등록되어 있으므로 직접 생성해야 한다.

```bash
# .env
DB_USERNAME=root
DB_PASSWORD=root1234
DB_ROOT_PW=root1234
DB_DATABASE_NAME=commerce
DB_TEST_DATABASE_NAME=testdb
DB_HOST=localhost
DB_PORT=3306
```

---

## 2. 실행 방법

### 방법 A. MySQL만 Docker로 실행 + 애플리케이션 로컬 실행 (권장)

개발 중에는 이 방식을 사용한다. 코드 변경 시 빠른 재기동이 가능하다.

**① MySQL 컨테이너 실행**

```bash
docker compose up -d mysql
```

**② 컨테이너 상태 확인**

```bash
docker compose ps
```

```
NAME      STATUS
mysql     running
```

**③ 애플리케이션 실행**

```bash
./gradlew bootRun
```

`local` 프로파일이 자동으로 활성화되며, `.env` 파일에서 DB 접속 정보를 읽는다.

---

### 방법 B. 전체 Docker Compose 실행

```bash
# 빌드 후 전체 실행 (app + mysql)
./gradlew build -x test
docker compose up -d
```

```bash
# 로그 확인
docker compose logs -f app
```

> `app` 서비스는 `dustjq1005/commerce:latest` 이미지를 사용한다.  
> 로컬 코드를 반영하려면 먼저 이미지를 직접 빌드해야 한다.

```bash
./gradlew build -x test
docker build -t dustjq1005/commerce:latest .
docker compose up -d
```

## 4. 테스트 실행

테스트는 **Testcontainers**를 사용하므로 Docker가 실행 중이어야 한다.  
별도의 DB 설정 없이 MySQL 컨테이너를 자동으로 띄우고 테스트 후 제거한다.

```bash
# 전체 테스트
./gradlew test

# 단일 클래스 테스트
./gradlew test --tests "com.mist.commerce.domain.product.repository.ProductRepositoryTest"

# 단일 메서드 테스트
./gradlew test --tests "com.mist.commerce.domain.product.repository.ProductRepositoryTest.findAll"

# 테스트 결과 리포트
# build/reports/tests/test/index.html
```

MySQL 연동이 필요한 테스트는 `MySqlContainerTestSupport`를 상속한다.

```java
@DataJpaTest
class ProductRepositoryTest extends MySqlContainerTestSupport {
    // ...
}
```

## 5. 프로파일 구성

| 프로파일 | 활성화 방법 | DB |
|----------|-------------|-----|
| `local` | 기본값 (`application.yml`에 설정) | Docker MySQL (3307) |
| `test` / `test-mysql` | `build.gradle`에서 자동 적용 | Testcontainers MySQL |

수동으로 프로파일을 지정해 실행할 경우:

```bash
./gradlew bootRun --args='--spring.profiles.active=local'
```

---

## 6. 종료 및 초기화

```bash
# 컨테이너 중지
docker compose down

# 컨테이너 + 볼륨(DB 데이터) 완전 삭제
docker compose down -v
```

> `docker compose down -v` 실행 시 `./data/` 디렉토리의 MySQL 데이터가 삭제된다.

## 7. 트러블슈팅

**MySQL 연결 실패 시**
- `.env` 파일이 프로젝트 루트에 있는지 확인한다.
- `docker compose ps`로 MySQL 컨테이너가 `running` 상태인지 확인한다.
- `DB_LOCALHOST_PORT`가 `3307`로 설정되어 있는지 확인한다.

**포트 충돌 시**
- 로컬 3307 포트가 이미 사용 중이라면 `docker-compose.yml`의 포트 매핑을 변경하고 `.env`의 `DB_LOCALHOST_PORT`도 맞춰 수정한다.

**Testcontainers 실행 실패 시**
- Docker Desktop이 실행 중인지 확인한다.
- `docker info` 명령어로 Docker 데몬 상태를 확인한다.
