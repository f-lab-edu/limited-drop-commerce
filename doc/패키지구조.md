# 패키지 구조

## 전략: 도메인(Domain) 중심 구조

기능별 레이어(`controller`, `service`, `repository`)로 나누는 대신, **도메인 단위로 묶는 구조**를 채택합니다.
규모가 커질 때 모듈 분리가 쉽고, 응집도가 높습니다.

---

## 단계 1 — 최상위 구조

```
com.mist.commerce
├── CommerceApplication.java       ← @SpringBootApplication (루트에 위치 필수)
│
├── common/                        ← 전역 공통 설정
├── config/                        ← Spring 설정 클래스
│
├── domain/                        ← 핵심 비즈니스 도메인
│   ├── user/
│   ├── product/
│   ├── order/
│   ├── reservation/
│   └── payment/
│
└── infra/                         ← 외부 연동 (PG사, 알림, 파일 등)
```

> **Spring Boot 공식 권고:** `@SpringBootApplication`이 있는 메인 클래스는 반드시 루트 패키지에 위치해야
> 컴포넌트 스캔이 전체 도메인에 적용됩니다.

---

## 단계 2 — 도메인 내부 레이어 구조

각 도메인은 동일한 내부 구조를 가집니다. `order` 도메인 예시:

```
domain/order/
├── controller/
│   └── OrderController.java          ← REST API 진입점
│
├── service/
│   ├── OrderService.java             ← 비즈니스 로직 (인터페이스)
│   └── OrderServiceImpl.java
│
├── repository/
│   └── OrderRepository.java          ← JPA Repository
│
├── entity/
│   └── Order.java                    ← @Entity
│
├── dto/
│   ├── OrderCreateRequest.java
│   ├── OrderResponse.java
│   └── OrderStatusUpdateRequest.java
│
└── exception/
    ├── OrderNotFoundException.java
    └── OrderAlreadyCanceledException.java
```

---

## 단계 3 — 핵심 도메인 관계 정의

```
user       ─┐
product    ─┤─→  reservation ─→  order ─→  payment
             └───────────────────────────────────────
```

| 도메인 | 역할 |
|--------|------|
| `user` | 회원 가입/인증, Security 연동 |
| `product` | 상품 정보, 재고 관리 |
| `reservation` | 한정 수량 선점/예약 (대기열 포함 가능) |
| `order` | 예약 → 주문 확정, 주문 상태 관리 |
| `payment` | PG 연동, 결제/취소/환불 |

---

## 단계 4 — `common` / `config` 내부

```
common/
├── response/
│   ├── ApiResponse.java              ← 공통 응답 포맷
│   └── ErrorResponse.java
├── exception/
│   └── GlobalExceptionHandler.java   ← @ControllerAdvice
├── entity/
│   └── BaseEntity.java               ← createdAt, updatedAt (@MappedSuperclass)
└── util/

config/
├── SecurityConfig.java
├── JpaConfig.java
├── WebMvcConfig.java                 ← CORS 설정
└── properties/
    └── AppProperties.java            ← @ConfigurationProperties
```

---

## 단계 5 — `infra` (외부 연동)

```
infra/
├── payment/
│   └── TossPaymentClient.java        ← PG사 HTTP 클라이언트
├── notification/
│   └── EmailNotificationSender.java
└── storage/
    └── S3FileUploader.java
```

> `infra`는 외부 시스템 의존성을 격리합니다. PG사를 교체해도 `domain/payment`의 비즈니스 로직은 변경 불필요.

---

## 최종 전체 구조

```
com.mist.commerce
├── CommerceApplication.java
├── common/
│   ├── response/
│   ├── exception/
│   ├── entity/BaseEntity.java
│   └── util/
├── config/
│   ├── SecurityConfig.java
│   ├── JpaConfig.java
│   ├── WebMvcConfig.java
│   └── properties/
├── domain/
│   ├── user/
│   │   └── controller/ service/ repository/ entity/ dto/ exception/
│   ├── product/
│   │   └── controller/ service/ repository/ entity/ dto/ exception/
│   ├── reservation/
│   │   └── controller/ service/ repository/ entity/ dto/ exception/
│   ├── order/
│   │   └── controller/ service/ repository/ entity/ dto/ exception/
│   └── payment/
│       └── controller/ service/ repository/ entity/ dto/ exception/
└── infra/
    ├── payment/
    ├── notification/
    └── storage/
```
